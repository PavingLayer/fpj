# fpj — File Projector

A tool for creating and managing flexible filesystem views
using layered overlays and bind mounts.

fpj lets you define **layers** (overlay filesystems with writable/locked states)
and **layouts** (ordered sequences of mount operations) that can be mounted,
unmounted, and restored atomically. All state is persisted in a local SQLite
database.

## Features

- Named layers with internal management of upper/work directories
- Layer stacking via flat chain resolution (child layers reference locked parents)
- Bind mount support interleaved with overlay layers
- Atomic mount/unmount with automatic rollback on failure
- Persistent layout definitions that survive reboots (`fpj restore`)
- Cross-platform: Linux (fuse-overlayfs), macOS (macFUSE),
  Windows (junction points + copy-based overlay)

## Installation

### From source

```
cargo install --path .
```

### Build without installing

```
cargo build --release
```

The binary will be at `target/release/fpj`.

## Quick start

```bash
# Create a layer backed by a directory
fpj layer create base-image --source /opt/platform --mount-point /workspace/merged

# Lock it so it can be used as a base for child layers
fpj layer lock base-image

# Create a child layer on top (writable by default)
fpj layer create customizations --source @base-image --mount-point /workspace/dev

# Create a layout that orchestrates the mount sequence
fpj layout create dev-env
fpj step add-layer dev-env --layer customizations
fpj step add-bind dev-env --source /home/user/config --target /workspace/dev/config

# Mount everything atomically
fpj mount dev-env

# Check status
fpj status dev-env

# Unmount when done
fpj unmount dev-env

# After reboot, restore all persisted layouts
fpj restore
```

## Shell completions

fpj supports dynamic shell completions that include layer names, layout names, and file paths. 
Completions are generated by the binary itself, so they always match the current version.

### Bash

```bash
echo 'source <(COMPLETE=bash fpj)' >> ~/.bashrc
```

### Zsh

```zsh
echo 'source <(COMPLETE=zsh fpj)' >> ~/.zshrc
```

### Fish

```fish
echo 'COMPLETE=fish fpj | source' >> ~/.config/fish/completions/fpj.fish
```

### PowerShell

```powershell
Add-Content $PROFILE '$env:COMPLETE = "powershell"; fpj | Out-String | Invoke-Expression; Remove-Item Env:\COMPLETE'
```

> **Note**: PowerShell on Windows requires the execution policy to be set to
> `RemoteSigned` at minimum for `$PROFILE` scripts to run.

After adding the line, restart your shell. Layer and layout names are completed
dynamically from the database, so `fpj layer show <TAB>` will suggest your
actual layer names.

## Documentation

Full documentation is built with [mdBook](https://rust-lang.github.io/mdBook/).

### Read locally

```bash
# Install mdbook if needed
cargo install mdbook

# Serve docs with live reload
mdbook serve docs --open
```

This starts a local server at `http://localhost:3000` with live reload (useful for development).

### Build static site

```bash
mdbook build docs
```

Output is written to `docs/book/`.

The documentation is also published to
[https://pavinglayer.github.io/fpj](https://pavinglayer.github.io/fpj) on every
push to `master`.

## Data directories

fpj stores its database and layer data under `~/.local/share/fpj/` (Linux),
`~/Library/Application Support/fpj/` (macOS), or `%APPDATA%\fpj\` (Windows):

```
<data_dir>/fpj/
├── fpj.db           # SQLite database (layouts, layers, steps)
└── layers/
    └── <layer-name>/
        ├── upper/   # Overlay upper directory
        └── work/    # Overlay work directory
```

Use `--db <path>` to override the database location.
